GitHub Pages 入门指南

GitHub Pages 是 GitHub 提供的免费静态网站托管服务。您可以使用它来托管个人博客、项目文档或任何静态网站。本文将详细介绍如何使用 GitHub Pages。

什么是 GitHub Pages？

GitHub Pages 允许您直接从 GitHub 仓库托管静态网站。它支持 HTML、CSS 和 JavaScript，并且完全免费。您的网站将可以通过 username.github.io 或自定义域名访问。

创建仓库

首先，您需要在 GitHub 上创建一个新仓库。有两种方式：

方式一：用户/组织站点

如果您想创建个人主页或组织主页，仓库名必须是 username.github.io 或 organization.github.io。这种方式下，网站将直接部署在根目录。

方式二：项目站点

对于项目文档或博客，您可以创建任意名称的仓库。然后需要在仓库设置中启用 GitHub Pages，并选择源分支（通常是 main 或 gh-pages）。

上传文件

将您的 HTML、CSS、JavaScript 等静态文件上传到仓库。确保有一个 index.html 文件作为网站的入口页面。

启用 GitHub Pages

  1. 进入您的仓库
  2. 点击 "Settings"（设置）
  3. 在左侧菜单中找到 "Pages"
  4. 在 "Source" 部分选择分支（通常是 main）
  5. 选择根目录（/）或 /docs 文件夹
  6. 点击 "Save"（保存）

访问您的网站

保存设置后，GitHub 会开始构建您的网站。几分钟后，您就可以通过以下地址访问：

  • 用户/组织站点：https://username.github.io
  • 项目站点：https://username.github.io/repository-name

使用自定义域名

如果您有自己的域名，可以在仓库根目录创建一个 CNAME 文件，内容为您的域名。然后在您的域名 DNS 设置中添加 CNAME 记录指向 username.github.io。

注意事项

  • 所有文件路径必须使用相对路径，不能使用绝对路径
  • GitHub Pages 只支持静态内容，不支持服务器端脚本（如 PHP）
  • 仓库必须是公开的（除非使用 GitHub Pro）
  • 每次推送代码后，网站会自动更新（可能需要几分钟）

总结

GitHub Pages 是一个简单而强大的工具，非常适合托管静态网站。通过本文的指南，您应该能够快速上手并部署自己的网站。如果您有任何问题，欢迎查看 GitHub Pages 的官方文档。
